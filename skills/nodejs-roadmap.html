Node.js Full Roadmap

Learn backend development using Node.js step by step.

1. Introduction to Node.js

Beginner

Description:
Node.js allows you to run JavaScript outside the browser.

• Used to build backend servers
• Built on Chrome’s V8 engine

It lets you use JavaScript for full-stack development.

Example: Run a simple Node script.


console.log("Hello from Node.js");

2. Node.js vs Browser

Beginner

Description:
JavaScript behaves differently in browser vs Node.

• Browser → handles UI (DOM, events)
• Node.js → handles server logic

Node does NOT have DOM or window object.

3. Installing Node & npm

Beginner

Description:
Node.js must be installed to run backend code.

• npm → installs packages (libraries)
• Used in every Node project

Example: Check installation.


node -v
npm -v

4. Event Loop

Intermediate

Description:
The event loop handles asynchronous operations.

• Executes tasks without blocking code
• Allows Node to handle multiple requests

This is why Node is fast and scalable.

5. Running JS Files

Beginner

Description:
You can run JavaScript files using Node.

• Used for scripts and backend apps

Example: Run file.


node app.js

6. Modules

Beginner

Description:
Modules allow you to split code into smaller parts.

• require() → import module
• Helps keep code organized

Example: Import module.


const fs = require("fs");

7. File System (fs)

Intermediate

Description:
The fs module lets you work with files.

• Read files
• Write files
• Delete files

Example: Write file.


const fs = require("fs");

fs.writeFileSync("test.txt", "Hello World");

8. HTTP Server

Intermediate

Description:
Node can create web servers using the http module.

• Handles client requests
• Sends responses back

Example: Basic server.


const http = require("http");

http.createServer((req, res) => {
  res.end("Hello Server");
}).listen(3000);

9. Promises

Intermediate

Description:
Promises handle async operations better than callbacks.

• then() → success
• catch() → error

10. Async / Await

Intermediate

Description:
Async/Await simplifies asynchronous code.

• Looks like normal code
• Easier to read and maintain

11. Promises

Intermediate

Description:
Promises are used to handle asynchronous operations in JavaScript.

Instead of using callbacks, promises provide a cleaner way to manage results of operations like file reading, API calls, or database queries.

• then() → runs when operation is successful
• catch() → handles errors

Promises help avoid deeply nested callback code (callback hell).

Example: Basic promise usage.


const promise = new Promise((resolve, reject) => {
  resolve("Success");
});

promise.then(data => console.log(data));

12. Async / Await

Intermediate

Description:
Async/Await is built on top of promises and makes asynchronous code easier to read.

• async → defines a function that returns a promise
• await → pauses execution until promise resolves

This makes code look like normal synchronous code while still being asynchronous.

Example: Using async/await.


async function fetchData() {
  return "Data received";
}

fetchData().then(console.log);

13. Creating REST API

Intermediate

Description:
A REST API allows communication between frontend and backend.

It uses HTTP methods: • GET → fetch data
• POST → create data
• PUT → update data
• DELETE → remove data

REST APIs are the backbone of modern web applications.

Example: Simple GET route.


app.get("/users", (req, res) => {
  res.send("User data");
});

14. Routing

Intermediate

Description:
Routing is used to handle different URLs in your application.

Each route represents a specific endpoint: • /users → user data
• /products → product data

It allows your server to respond differently based on request paths.

15. Request & Response

Intermediate

Description:
Every HTTP interaction has two parts:

• Request (req) → data sent by client
• Response (res) → data sent back by server

Understanding req and res is essential to control how your API behaves.

16. JSON Handling

Beginner

Description:
JSON (JavaScript Object Notation) is the standard data format used in APIs.

• Used to send and receive data
• Lightweight and easy to parse

Most APIs communicate using JSON.

17. Express.js

Intermediate

Description:
Express is a framework built on Node.js that simplifies backend development.

It provides tools for: • Routing
• Middleware
• Request handling

It removes the complexity of using raw Node.js.

Example: Basic Express server.


const express = require("express");
const app = express();

app.get("/", (req, res) => {
  res.send("Hello Express");
});

app.listen(3000);

18. Middleware

Advanced

Description:
Middleware functions run between request and response.

They are used for: • Logging
• Authentication
• Data processing

Middleware is one of the most powerful features of Express.

19. Error Handling

Advanced

Description:
Error handling ensures your application does not crash unexpectedly.

• Catch runtime errors
• Send proper error responses

Proper error handling improves debugging and user experience.

20. Express Router

Advanced

Description:
Express Router helps organize routes into separate files.

• Keeps code clean
• Makes large apps manageable

Essential for scalable backend applications.

21. MongoDB with Node.js

Advanced

Description:
MongoDB is used with Node.js to store application data.

Node.js handles server logic, while MongoDB stores and retrieves data.

• Used for storing users, products, etc.
• Connect using MongoDB driver or Mongoose

This is the foundation of full-stack applications.

Example: Connect to MongoDB.


const mongoose = require("mongoose");

mongoose.connect("mongodb://localhost:27017/mydb")
  .then(() => console.log("Connected"))
  .catch(err => console.log(err));

22. Mongoose (Schema & Models)

Advanced

Description:
Mongoose adds structure to MongoDB by using schemas.

• Schema → defines data structure
• Model → used to interact with database

It makes database operations easier and more organized.

Example: Create a schema.


const userSchema = new mongoose.Schema({
  name: String,
  age: Number
});

const User = mongoose.model("User", userSchema);

23. CRUD Operations

Advanced

Description:
CRUD stands for Create, Read, Update, Delete.

These operations are used to manage data in the database:

• Create → add new data
• Read → fetch data
• Update → modify data
• Delete → remove data

Every backend application depends on CRUD.

Example: Save data.


const user = new User({ name: "Vijay", age: 20 });
user.save();

24. Authentication (Login/Signup)

Advanced

Description:
Authentication verifies the identity of users.

• Signup → create account
• Login → verify user credentials

It is required in almost every real-world application.

25. Password Hashing (bcrypt)

Advanced

Description:
Passwords should never be stored as plain text.

• Hash passwords before saving
• Compare hashed values during login

This protects user data from being exposed.

Example: Hash password.


const bcrypt = require("bcrypt");

const hashed = await bcrypt.hash("password123", 10);

26. JWT Authentication

Advanced

Description:
JWT (JSON Web Token) is used for user authentication.

• Generates a token after login
• Token is sent with requests

Used to maintain user sessions in APIs.

Example: Generate token.


const jwt = require("jsonwebtoken");

const token = jwt.sign({ id: 1 }, "secret");

27. Protected Routes

Advanced

Description:
Protected routes restrict access to authorized users only.

• Check token before allowing access
• Used for dashboards, private data

Ensures security in backend applications.

28. MVC Architecture

Advanced

Description:
MVC organizes backend code into three parts:

• Model → data & database
• View → UI (frontend)
• Controller → logic

Helps maintain clean and scalable code.

29. Environment Variables

Advanced

Description:
Environment variables store sensitive data securely.

• API keys
• Database URLs

Prevents exposing secrets in code.

Example: Use dotenv.


require("dotenv").config();

console.log(process.env.PORT);

30. Deployment

Advanced

Description:
Deployment makes your backend available online.

• Host server (Render, Railway, AWS)
• Connect cloud database (MongoDB Atlas)

This is the final step to make your app usable.
