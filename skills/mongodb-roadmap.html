MongoDB Full Roadmap

Learn MongoDB from basics to advanced concepts like CRUD operations, aggregation, indexing, and real-world backend integration. Click Show Code to view examples in a popup.

1. Introduction to MongoDB

Beginner

Description:
MongoDB is a NoSQL database that stores data in flexible, JSON-like documents. Unlike traditional databases, it does not use tables and rows.

It is designed for: - High performance - Scalability - Flexible schema

Example: Storing user data as documents instead of rows.

Example Question: Create a simple document structure.


{
  name: "Vijay",
  age: 20,
  skills: ["C++", "JavaScript"]
}

2. NoSQL vs SQL

Beginner

Description:
SQL databases use tables and fixed schemas. NoSQL databases like MongoDB use flexible documents.

SQL: - Structured - Fixed schema

NoSQL: - Flexible schema - Better for large-scale and unstructured data

Example Question: Compare SQL row with MongoDB document.


-- SQL (Row)
id | name | age
1  | Vijay | 20

// MongoDB (Document)
{
  "_id": 1,
  "name": "Vijay",
  "age": 20
}

3. Installing MongoDB & Compass

Beginner

Description:
MongoDB can be installed locally or used via MongoDB Atlas (cloud).

Tools: - MongoDB Server → runs database - MongoDB Compass → GUI tool to manage data - mongosh → command-line shell

Example Question: Start MongoDB shell.


mongosh

4. MongoDB Architecture

Beginner

Description:
MongoDB organizes data in a hierarchical structure:

- Database → contains collections - Collection → contains documents - Document → actual data (JSON-like)

Example: A "users" collection inside a "college" database.

Example Question: Show database structure.


use college

db.users.insertOne({
  name: "Vijay",
  age: 20
})

5. BSON & JSON Structure

Beginner

Description:
MongoDB stores data in BSON (Binary JSON).

JSON: - Human readable

BSON: - Faster storage - Supports more data types (Date, ObjectId)

You write JSON, MongoDB stores BSON internally.

Example Question: Insert document with ObjectId.


db.users.insertOne({
  name: "Vijay",
  createdAt: new Date()
})

6. MongoDB Shell (mongosh)

Beginner

Description:
mongosh is the interactive shell used to interact with MongoDB.

You can: - Create databases - Insert data - Run queries

It is the fastest way to test MongoDB commands.

Example Question: Show all databases.


show dbs

7. Insert Documents

Beginner

Description:
Insert data into a collection.

• insertOne() → insert a single document
• insertMany() → insert multiple documents

Used when adding new records.

Example: Insert a user document.


db.users.insertOne({
  name: "Vijay",
  age: 20
})

8. Read Documents

Beginner

Description:
Retrieve data from collections.

• find() → get multiple documents
• findOne() → get a single document

Can also filter results using conditions.

Example: Find users with age 20.


db.users.find({ age: 20 })

9. Query Operators

Intermediate

Description:
Used to filter data with conditions.

• $gt → greater than
• $lt → less than
• $in → match values in array
• $and / $or → combine conditions

Example: Find users age > 18.


db.users.find({
  age: { $gt: 18 }
})

10. Update Documents

Intermediate

Description:
Modify existing documents.

• updateOne() → update one document
• updateMany() → update multiple
• $set → change value
• $inc → increase value

Example: Update user age.


db.users.updateOne(
  { name: "Vijay" },
  { $set: { age: 21 } }
)

11. Delete Documents

Intermediate

Description:
Remove documents from a collection.

• deleteOne() → remove one document
• deleteMany() → remove multiple

Use carefully — deleted data cannot be recovered.

Example: Delete a user.


db.users.deleteOne({ name: "Vijay" })

12. Schema Design

Intermediate

Description:
Schema design defines how your data is structured in MongoDB.

• MongoDB is flexible (no fixed schema)
• But good structure is still important

Goal: • Keep data simple, efficient, and scalable

Example: Store user with nested address.


db.users.insertOne({
  name: "Vijay",
  address: {
    city: "Hyderabad",
    pincode: 500001
  }
})

13. Embedded vs Referencing

Intermediate

Description:
Two ways to store related data.

• Embedded → store data inside document
• Referencing → store data in separate collections

Use embedded → for fast reads
Use referencing → for large or reusable data

Example: Embedded user orders.


db.users.insertOne({
  name: "Vijay",
  orders: [
    { item: "Laptop", price: 50000 },
    { item: "Mouse", price: 500 }
  ]
})

14. Data Relationships

Intermediate

Description:
Defines how data is connected.

• One-to-One → user ↔ profile
• One-to-Many → user → orders
• Many-to-Many → students ↔ courses

Choose structure based on use-case.

Example: One-to-many relationship.


db.orders.insertOne({
  userId: 1,
  item: "Laptop"
})

15. Data Validation

Intermediate

Description:
Ensures only valid data is stored.

• Prevents wrong or incomplete data
• Adds basic rules to collections

Helps maintain data quality in real applications.

Example: Validate age field.


db.createCollection("users", {
  validator: {
    $jsonSchema: {
      bsonType: "object",
      required: ["name", "age"],
      properties: {
        age: {
          bsonType: "int",
          minimum: 18
        }
      }
    }
  }
})

16. Projection

Intermediate

Description:
Projection is used to return only required fields.

• Reduces data transfer
• Improves performance

Instead of fetching full documents, you select specific fields.

Example: Show only name field.


db.users.find(
  {},
  { name: 1, _id: 0 }
)

17. Sorting & Limiting

Intermediate

Description:
Used to control output results.

• sort() → order data (asc/desc)
• limit() → restrict number of results
• skip() → skip records (pagination)

Example: Get top 2 users by age.


db.users.find()
  .sort({ age: -1 })
  .limit(2)

18. Indexing

Advanced

Description:
Indexes improve query performance.

• Without index → full scan (slow)
• With index → fast lookup

Types: • Single field
• Compound index

Example: Create index on age.


db.users.createIndex({ age: 1 })

19. Text Search & Regex

Advanced

Description:
Used for searching text data.

• $text → full-text search
• $regex → pattern matching

Useful for search features (like search bar).

Example: Find users with name starting with "V".


db.users.find({
  name: { $regex: "^V" }
})

20. Aggregation Pipeline

Advanced

Description:
Aggregation processes data step-by-step like a pipeline.

• Each stage transforms data
• Output of one stage → input of next

Used for: • Data analysis
• Reports
• Complex queries

Example: Basic pipeline structure.


db.users.aggregate([
  { $match: { age: { $gt: 18 } } }
])

21. $match, $group, $project

Advanced

Description:
Core aggregation stages.

• $match → filter data
• $group → group data (like SQL GROUP BY)
• $project → reshape fields

Example: Count users by age.


db.users.aggregate([
  {
    $group: {
      _id: "$age",
      count: { $sum: 1 }
    }
  }
])

22. $sort, $limit, $skip

Intermediate

Description:
Control output in aggregation.

• $sort → order data
• $limit → restrict results
• $skip → pagination

Example: Top 2 users by age.


db.users.aggregate([
  { $sort: { age: -1 } },
  { $limit: 2 }
])

23. $lookup (Join)

Advanced

Description:
Used to join data from another collection.

• Works like SQL JOIN
• Combines related documents

Used when data is stored separately.

Example: Join users with orders.


db.users.aggregate([
  {
    $lookup: {
      from: "orders",
      localField: "_id",
      foreignField: "userId",
      as: "orders"
    }
  }
])

24. $unwind

Advanced

Description:
Breaks array into separate documents.

• Converts each array item into a document
• Useful for processing nested data

Example: Unwind orders array.


db.users.aggregate([
  { $unwind: "$orders" }
])

25. Real-world Aggregation

Advanced

Description:
Combine multiple stages to solve real problems.

Example: • Filter → Group → Sort
• Used in dashboards and analytics

Example: Count users age > 18 and sort.


db.users.aggregate([
  { $match: { age: { $gt: 18 } } },
  {
    $group: {
      _id: "$age",
      count: { $sum: 1 }
    }
  },
  { $sort: { count: -1 } }
])

26. Index Optimization

Advanced

Description:
Index optimization improves query speed.

• Use indexes on frequently searched fields
• Avoid too many indexes (slows writes)

Goal: • Balance read speed and write performance

Example: Create compound index.


db.users.createIndex({ name: 1, age: -1 })

27. Query Performance Analysis

Advanced

Description:
Analyze how queries are executed.

• explain() shows query plan
• Helps identify slow queries

Used to improve performance.

Example: Analyze a query.


db.users.find({ age: 20 }).explain("executionStats")

28. Sharding

Advanced

Description:
Sharding splits data across multiple servers.

• Used for very large datasets
• Improves scalability

Data is distributed using shard keys.

Example: Enable sharding.


sh.enableSharding("mydb")

29. Replication

Advanced

Description:
Replication creates multiple copies of data.

• Ensures high availability
• Provides backup if server fails

Uses replica sets (primary + secondary).

Example: Initiate replica set.


rs.initiate()

30. Authentication & Authorization

Advanced

Description:
Controls who can access the database.

• Authentication → verify user identity
• Authorization → control permissions

Prevents unauthorized access to data.

Example: Create a database user.


db.createUser({
  user: "admin",
  pwd: "password123",
  roles: [{ role: "readWrite", db: "mydb" }]
})

31. Role-Based Access Control

Advanced

Description:
Assign permissions using roles.

• read → only read data
• readWrite → read + write
• admin → full control

Helps manage user access efficiently.

Example: Assign read-only role.


db.createUser({
  user: "viewer",
  pwd: "1234",
  roles: [{ role: "read", db: "mydb" }]
})

32. Data Backup & Restore

Advanced

Description:
Protect data from loss.

• Backup → save database copy
• Restore → recover data

Essential for production systems.

Example: Backup database.


mongodump --db mydb --out backup/

33. Best Practices

Advanced

Description:
Follow best practices for production systems.

• Use indexes wisely
• Avoid large documents
• Validate data
• Use proper schema design

Helps build scalable and maintainable apps.

Example: Create indexed collection.


db.users.createIndex({ email: 1 }, { unique: true })

38. Transactions

Advanced

Description:
Transactions allow multiple operations to run safely as one unit.

• Either all operations succeed
• Or none are applied (rollback)

Used in critical operations like payments.

Example: Start a transaction.


const session = await mongoose.startSession();

session.startTransaction();

// operations here

await session.commitTransaction();

39. Change Streams

Advanced

Description:
Change streams listen to real-time database updates.

• Detect inserts, updates, deletes
• Used for live apps (notifications, chats)

Example: Watch collection changes.


const changeStream = db.collection("users").watch();

changeStream.on("change", (change) => {
  console.log(change);
});

40. GridFS

Advanced

Description:
GridFS is used to store large files in MongoDB.

• Splits files into chunks
• Stores them efficiently

Used for images, videos, documents.

Example: Upload file (concept).


// Using GridFS (concept)
const bucket = new mongoose.mongo.GridFSBucket(db);

const uploadStream = bucket.openUploadStream("file.txt");

41. MongoDB Atlas (Cloud)

Advanced

Description:
MongoDB Atlas is a cloud-hosted database service.

• No local setup required
• Easy scaling and backups

Used in real-world applications.

Example: Connect to Atlas.


mongoose.connect("your-atlas-connection-string")
  .then(() => console.log("Connected"))
  .catch(err => console.log(err));

42. Build REST API with MongoDB

Advanced

Description:
Use MongoDB in backend APIs to handle data.

• GET → fetch data
• POST → create data
• PUT → update data
• DELETE → remove data

This connects your database to real applications.

Example: Fetch all users.


app.get("/users", async (req, res) => {
  const users = await User.find();
  res.json(users);
});

43. Authentication System

Advanced

Description:
Build login and signup functionality.

• Store user credentials securely
• Hash passwords using bcrypt

Required for any real-world application.

Example: Hash a password.


const bcrypt = require("bcrypt");

const hashed = await bcrypt.hash("password123", 10);

44. Scalable Backend Design

Advanced

Description:
Design systems that can handle growth.

• Use proper schema design
• Optimize queries and indexes
• Avoid unnecessary duplication

Ensures performance as users increase.

Example: Add index for faster queries.


db.users.createIndex({ email: 1 })

45. Deployment

Advanced

Description:
Deploy your application online.

• Use MongoDB Atlas (cloud database)

Makes your app accessible to real users.

Example: Connect to Atlas.


mongoose.connect(process.env.MONGO_URI)
