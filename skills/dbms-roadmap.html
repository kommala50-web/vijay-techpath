DBMS Full Roadmap

Learn Database Management Systems from fundamentals to advanced topics including SQL, normalization, transactions, indexing, and performance. Click Show Code to view examples.

1. Introduction to DBMS

Beginner

Description:
Database Management Systems (DBMS) are software systems designed to store, manage, and retrieve data efficiently. They provide mechanisms for data integrity, security, concurrency, and backup. Types include relational (RDBMS), NoSQL, hierarchical, and network databases. 


// Example: Creating a table in SQL
CREATE TABLE Students (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

2. ER Model & Design

Beginner

Description:
Entity-Relationship (ER) modeling is a conceptual design tool to represent data and relationships. ER diagrams consist of entities, attributes, and relationships. Translating ER models to relational schemas is a key step in database design. 


CREATE TABLE Student (
  student_id INT PRIMARY KEY,
  name VARCHAR(50)
);

CREATE TABLE Course (
  course_id INT PRIMARY KEY,
  title VARCHAR(50)
);

CREATE TABLE Enrollment (
  student_id INT,
  course_id INT,
  PRIMARY KEY(student_id, course_id),
  FOREIGN KEY(student_id) REFERENCES Student(student_id),
  FOREIGN KEY(course_id) REFERENCES Course(course_id)
);

3. Normalization

Intermediate

Description:
Normalization organizes tables to reduce redundancy and improve data integrity. Normal forms include 1NF, 2NF, 3NF, BCNF, 4NF, and 5NF. 


-- 1NF requires atomic values
-- Separate rows instead of multiple values in one column

4. SQL Queries

Beginner

Description:
SQL is used to interact with relational databases. Operations include SELECT, INSERT, UPDATE, DELETE, JOINs, GROUP BY, HAVING, and subqueries. 


SELECT name, age FROM Students WHERE age > 20;

SELECT S.name, C.title
FROM Student S
JOIN Enrollment E ON S.student_id = E.student_id
JOIN Course C ON E.course_id = C.course_id;

5. Transactions & ACID

Intermediate

Description:
Transactions are sequences of operations executed as a single unit. ACID properties ensure reliability and consistency. 


BEGIN TRANSACTION;
UPDATE Accounts SET balance = balance - 100 WHERE id = 1;
UPDATE Accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

6. Indexing & Performance

Intermediate

Description:
Indexes improve query performance by enabling faster searches at the cost of additional storage and write overhead. 


CREATE INDEX idx_student_name ON Student(name);

7. Views, Stored Procedures & Triggers

Intermediate

Description:
Views are virtual tables created from SQL queries and do not store data physically. Stored procedures are reusable blocks of SQL logic stored in the database. Triggers are special procedures that automatically execute in response to INSERT, UPDATE, or DELETE operations on a table.

Example Question: Create a view and a trigger for logging updates.


-- View example
CREATE VIEW AdultStudents AS
SELECT * FROM Students WHERE age >= 18;

-- Trigger example
CREATE TRIGGER log_update
AFTER UPDATE ON Students
FOR EACH ROW
INSERT INTO StudentLog(student_id, change_time)
VALUES (NEW.id, NOW());

8. Concurrency Control

Intermediate

Description:
Concurrency control ensures that multiple transactions execute safely without corrupting data. It prevents problems like lost updates, dirty reads, and inconsistent data. Common techniques include locking, timestamp ordering, and optimistic concurrency control.

Example Question: How do you prevent two users from updating the same record simultaneously?


-- Lock a row before updating
SELECT * FROM Accounts WHERE id = 1 FOR UPDATE;

9. Deadlocks & Resolution

Intermediate

Description:
A deadlock occurs when two or more transactions wait indefinitely for resources held by each other. DBMS detects deadlocks using wait-for graphs or timeouts and resolves them by aborting one transaction.

Example Question: How does a DBMS resolve deadlocks?


SET DEADLOCK_PRIORITY LOW;

BEGIN TRANSACTION;
UPDATE Accounts SET balance = balance - 100 WHERE id = 1;
UPDATE Accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

10. Recovery & Backup

Intermediate

Description:
Recovery mechanisms restore databases after failures such as crashes or power loss. Techniques include logging (UNDO/REDO), checkpoints, and backups. Backups can be full, incremental, or differential.

Example Question: How do you back up and restore a database?


-- MySQL backup
mysqldump -u root -p dbname > backup.sql

-- Restore
mysql -u root -p dbname < backup.sql

11. Locking Mechanisms

Advanced

Description:
Locks control access to database resources. Shared locks allow reading, exclusive locks allow writing, and intent locks indicate future locking plans. Proper locking ensures data consistency in concurrent environments.

Example Question: Difference between shared and exclusive locks?


-- Shared lock
SELECT * FROM Accounts WHERE id = 1 LOCK IN SHARE MODE;

-- Exclusive lock
SELECT * FROM Accounts WHERE id = 1 FOR UPDATE;

12. Two-Phase Locking (2PL)

Advanced

Description:
Two-Phase Locking ensures serializability of transactions. In the growing phase, locks are acquired. In the shrinking phase, locks are released. No new locks can be acquired after releasing any lock.

Example Question: How does 2PL ensure consistency?


BEGIN TRANSACTION;

-- Growing phase
SELECT * FROM Accounts WHERE id = 1 FOR UPDATE;
SELECT * FROM Accounts WHERE id = 2 FOR UPDATE;

UPDATE Accounts SET balance = balance - 100 WHERE id = 1;
UPDATE Accounts SET balance = balance + 100 WHERE id = 2;

-- Shrinking phase
COMMIT;

13. Isolation Levels

Advanced

Description:
Isolation levels define how transactions interact with each other and prevent concurrency anomalies like dirty reads, non-repeatable reads, and phantom reads. Common levels are Read Uncommitted, Read Committed, Repeatable Read, and Serializable.

Example Question: Which isolation level prevents dirty reads?


SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

14. Replication

Advanced

Description:
Replication copies data across multiple database servers to improve availability, fault tolerance, and read performance. Common types include master-slave and master-master replication.

Example Question: How does replication improve availability?


CHANGE MASTER TO
MASTER_HOST='master_host',
MASTER_USER='replica_user',
MASTER_PASSWORD='password';
START SLAVE;

15. Sharding

Advanced

Description:
Sharding distributes large datasets across multiple databases or servers. Each shard contains a subset of data, improving scalability and performance in large-scale systems.

Example Question: How would you shard a global user database?


-- Example sharding strategy

16. Indexing Strategies

Advanced

Description:
Indexing strategies optimize query performance. Types include single-column, composite, unique, clustered, non-clustered, full-text, and covering indexes.

Example Question: How do composite indexes improve performance?


CREATE INDEX idx_student_name_age
ON Student(name, age);

17. Materialized Views

Advanced

Description:
Materialized views store query results physically instead of computing them dynamically. They significantly improve performance for complex analytical queries but require refresh mechanisms.

Example Question: When should you use a materialized view?


CREATE MATERIALIZED VIEW StudentSummary AS
SELECT student_id, COUNT(course_id) total_courses
FROM Enrollment
GROUP BY student_id;

18. Partitioning

Advanced

Description:
Partitioning divides large tables into smaller parts for better performance and manageability. Common types include range, list, hash, and composite partitioning.

Example Question: How does range partitioning work?


CREATE TABLE Sales (
  id INT,
  sale_date DATE
)
PARTITION BY RANGE (YEAR(sale_date)) (
  PARTITION p2023 VALUES LESS THAN (2024),
  PARTITION p2024 VALUES LESS THAN (2025)
);

19. NoSQL Basics

Advanced

Description:
NoSQL databases are designed for scalability and flexible schemas. Types include document stores, key-value stores, column-family, and graph databases.

Example Question: When should NoSQL be preferred over RDBMS?


db.users.insertOne({

  age: 22,
  preferences: { theme: "dark" }

20. Big Data Integration

Advanced

Description:
Modern DBMS integrates with big data systems like Hadoop and Spark using ETL pipelines, streaming ingestion, and analytics engines.

MongoDB supports advanced operations such as aggregation pipelines, indexing, replication, and sharding for large-scale applications.

Advanced

Description:
Advanced DBMS topics include distributed databases, OLAP, data warehousing, in-memory databases, database security, encryption, and distributed transactions.

Example Question: How do distributed databases maintain consistency?
