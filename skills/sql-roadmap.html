SQL Full Roadmap

Learn SQL from basics to advanced database querying.

1. Introduction to SQL

Beginner

Description:
SQL (Structured Query Language) is used to interact with databases. It allows you to store, retrieve, update, and delete data efficiently, and is widely used in backend systems, analytics, and data management. 


SELECT "Hello SQL";

2. Database & Table Basics

Beginner

Description:
A database stores data in structured form, and tables organize this data into rows and columns. Each table represents an entity, and each row represents a record. 


CREATE DATABASE mydb;

CREATE TABLE users (
  id INT,
  name VARCHAR(50)
);

3. SELECT Query

Beginner

Description:
SELECT is used to retrieve data from a table. You can select specific columns or all columns using * depending on your requirement. 


SELECT * FROM users;
SELECT name FROM users;

4. WHERE Clause

Beginner

Description:
The WHERE clause filters records based on conditions. It allows retrieving only the data that matches specific criteria. 


SELECT * FROM users WHERE age > 18;

5. INSERT Data

Beginner

Description:
INSERT is used to add new records into a table. You must specify values corresponding to each column. 


INSERT INTO users (id, name) VALUES (1, 'Ramesh');

6. UPDATE Data

Beginner

Description:
UPDATE modifies existing records in a table. It is usually combined with WHERE to avoid updating all rows unintentionally. 


UPDATE users SET name = 'Sita' WHERE id = 1;

7. DELETE Data

Beginner

Description:
DELETE removes records from a table. Using WHERE is important to avoid deleting all data accidentally. 


DELETE FROM users WHERE id = 1;

8. ORDER BY

Beginner

Description:
ORDER BY is used to sort the result set in ascending or descending order based on one or more columns. 


SELECT * FROM users ORDER BY name ASC;

9. LIMIT Clause

Beginner

Description:
LIMIT restricts the number of records returned by a query. It is useful when dealing with large datasets. 


SELECT * FROM users LIMIT 5;

10. Aggregate Functions

Intermediate

Description:
Aggregate functions perform calculations on multiple rows and return a single result. Common functions include COUNT, SUM, AVG, MIN, and MAX. 


SELECT COUNT(*) FROM users;
SELECT AVG(age) FROM users;

11. GROUP BY

Intermediate

Description:
GROUP BY is used to group rows that have the same values in specified columns. It is commonly used with aggregate functions to perform calculations on grouped data. 


SELECT department, COUNT(*) 
FROM employees 
GROUP BY department;

12. HAVING Clause

Intermediate

Description:
HAVING is used to filter grouped data after applying GROUP BY. Unlike WHERE, it works on aggregated results. 


SELECT department, COUNT(*) 
FROM employees 
GROUP BY department
HAVING COUNT(*) > 5;

13. INNER JOIN

Intermediate

Description:
INNER JOIN combines rows from two tables based on a matching condition. It returns only the rows where the condition is satisfied in both tables. 


SELECT users.name, orders.amount
FROM users
INNER JOIN orders
ON users.id = orders.user_id;

14. LEFT JOIN

Intermediate

Description:
LEFT JOIN returns all records from the left table and matched records from the right table. If no match exists, NULL values are returned for the right table. 


SELECT users.name, orders.amount
FROM users
LEFT JOIN orders
ON users.id = orders.user_id;

15. RIGHT JOIN

Intermediate

Description:
RIGHT JOIN returns all records from the right table and matching records from the left table. If no match exists, NULL values appear for the left table. 


SELECT users.name, orders.amount
FROM users
RIGHT JOIN orders
ON users.id = orders.user_id;

16. FULL JOIN

Intermediate

Description:
FULL JOIN returns all records when there is a match in either table. If there is no match, NULL values are returned for missing sides. 


SELECT users.name, orders.amount
FROM users
FULL JOIN orders
ON users.id = orders.user_id;

17. Subqueries

Intermediate

Description:
A subquery is a query inside another query. It is used to perform operations that depend on the result of another query. 


SELECT name 
FROM users 
WHERE id IN (
  SELECT user_id FROM orders
);

18. Views

Intermediate

Description:
A view is a virtual table based on a query. It simplifies complex queries and improves security by restricting direct access to data. 


CREATE VIEW user_orders AS
SELECT users.name, orders.amount
FROM users
JOIN orders ON users.id = orders.user_id;

19. Indexes

Advanced

Description:
Indexes improve query performance by allowing faster data retrieval. They are especially useful for large datasets but should be used carefully as they affect write performance. 


CREATE INDEX idx_name ON users(name);

20. Constraints

Advanced

Description:
Constraints enforce rules on table data to maintain integrity. Common constraints include PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, and CHECK. 


CREATE TABLE users (
  id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  email VARCHAR(100) UNIQUE
);

21. Transactions

Advanced

Description:
Transactions ensure a group of SQL operations are executed safely. They follow ACID properties (Atomicity, Consistency, Isolation, Durability) to maintain data integrity in critical operations like banking systems. 


BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

COMMIT;

22. Rollback

Advanced

Description:
ROLLBACK is used to undo changes made during a transaction. It ensures that if an error occurs, the database can return to its previous consistent state. 


BEGIN;

UPDATE users SET name = 'Error';

ROLLBACK;

23. Stored Procedures

Advanced

Description:
Stored procedures are pre-written SQL programs stored in the database. They improve performance, reuse logic, and reduce redundancy in complex operations. 


CREATE PROCEDURE getUsers()
BEGIN
  SELECT * FROM users;
END;

24. Triggers

Advanced

Description:
Triggers automatically execute when certain events occur on a table, such as INSERT, UPDATE, or DELETE. They are useful for enforcing rules and maintaining audit logs. 


CREATE TRIGGER before_insert
BEFORE INSERT ON users
FOR EACH ROW
SET NEW.name = UPPER(NEW.name);

25. Normalization

Advanced

Description:
Normalization is the process of organizing data to reduce redundancy and improve data integrity. It involves dividing tables into smaller related tables using rules called normal forms. 


-- Example: splitting user and orders tables
users(id, name)
orders(id, user_id, amount)

26. Denormalization

Advanced

Description:
Denormalization combines tables to improve read performance. It reduces joins but increases redundancy, and is commonly used in high-performance systems. 


-- Example: storing user name directly in orders table
orders(id, user_name, amount)

27. ACID Properties

Advanced

Description:
ACID properties define how transactions behave to ensure reliability: Atomicity (all or nothing), Consistency (valid state), Isolation (independent execution), and Durability (permanent storage). 


-- Conceptual example (no direct SQL)
-- ACID ensures safe transactions

28. Query Optimization

Advanced

Description:
Query optimization improves performance by reducing execution time. Techniques include indexing, avoiding unnecessary joins, and writing efficient queries. 


EXPLAIN SELECT * FROM users WHERE name = 'Ramesh';

29. Database Design

Advanced

Description:
Database design involves planning tables, relationships, and constraints. A good design ensures scalability, performance, and maintainability of applications. 


-- Example schema design
users(id, name)
orders(id, user_id)

30. Real-world SQL Projects

Advanced

Description:
Building projects like inventory systems, banking systems, or analytics dashboards helps apply SQL concepts in real-world scenarios and strengthens understanding. 


-- Example: fetch user orders
SELECT users.name, orders.amount
FROM users
JOIN orders ON users.id = orders.user_id;
